# How to Add a New Benchmark Plot

## Overview

Adding a new plot requires changes in **3 files**, following 4 steps.
The architecture ensures every plot automatically gets:

- Style presets (font sizes, margins, legend placement)
- Separate HTML/PNG export (controlled via `save_html` / `save_png`)
- A toggle flag to enable/disable the plot

## Architecture at a Glance

```
main.R                  → show_BmX_* flag (on/off toggle)
plot_styling.R          → style preset (fonts, margins, export sizes)
benchmark_plots.R       → plot function + wiring in run_all_benchmarks()
```

Render pipeline (called automatically by `render_benchmark_plot()`):

```
build_render_config()   → merge global config + preset export keys
build_styled_plots()    → apply style to HTML/PNG plotly objects
render_styled_plot()    → print + save + error handling
```

## Step 1: Add a Toggle Flag (`main.R`)

In `get_default_config()`, add a boolean flag:

```r
show_Bm1_konvergenz        = TRUE,
show_Bm1_konvergenz_single = FALSE,
# ...
show_BmX_my_new_plot       = TRUE,     # ← new
```

The benchmark function checks this flag as its first line.

## Step 2: Add a Style Preset (`plot_styling.R`)

In `get_style_profiles()`, add an entry to the returned list.
Every preset inherits from either `native` or `ioh` base:

| Base | Use for |
|------|---------|
| `native` | Plots you build directly with `plotly::plot_ly()` |
| `ioh` | Plots generated by `IOHanalyzer::Plot.*` (multi-subplot with annotations) |

**Minimal preset (inherits all defaults, only overrides export size):**

```r
my_new_preset = preset(native, list(
  png_width  = 1400,
  png_height = 1000
))
```

**Available override fields:**

| Field | Description |
|-------|-------------|
| `font_size` | General font size |
| `title_font_size` | Plot title font size |
| `axis_title_size` | Axis label font size |
| `axis_tick_size` | Tick label font size |
| `legend_font_size` | Legend text font size |
| `html_font_scale` | Scale factor for HTML output |
| `png_font_scale` | Scale factor for PNG export |
| `top_margin_title` | Top margin when title is shown |
| `bottom_margin` | Bottom margin |
| `left_margin` / `right_margin` | Side margins |
| `png_width` / `png_height` | PNG export dimensions (px) |
| `annotation_axis_title_size` | IOH annotation axis titles (ioh base only) |
| `annotation_subplot_size` | IOH subplot annotations (ioh base only) |

> **Tip:** A typo in the preset name triggers a warning at runtime listing all
> available presets — so misspellings are easy to catch.

## Step 3: Write the Plot Function (`benchmark_plots.R`)

Place the function in the appropriate section. The simplest pattern
(based on `benchmark_3_robustness`):

```r
benchmark_X_my_new_plot <- function(plot_data, cfg) {
  if (!isTRUE(cfg$show_BmX_my_new_plot)) return(invisible(NULL))

  # --- Build your plotly object ---
  p <- plotly::plot_ly(
    data = my_data,
    x = ~x_col, y = ~y_col,
    type = "scatter", mode = "lines"
  )

  p <- p |> plotly::layout(
    title = list(text = "My Title"),
    xaxis = list(title = "X Axis"),
    yaxis = list(title = "Y Axis")
  )

  # --- Render (style + save is automatic) ---
  render_benchmark_plot(
    p,
    cfg,
    "plot_my_new_plot",              # file name for export
    plot_type = "my_new_preset",     # must match key in get_style_profiles()
    log_label = "[Plot] My new plot"
  )
}
```

**Available data objects** (pick only what you need):

| Parameter | Contains |
|-----------|----------|
| `dsl_plot_base` | Raw DataSetList (budget-trimmed, algorithm-filtered) |
| `analysis_data` | Per-problem data: `analysis_data[["1"]]$abs`, `$rel`, `$global_min` |
| `plot_data` | Aggregated datasets: `$best_run_dataset`, `$all_runs_dataset`, `$fv_abs`, `$fv_rel` |
| `selected_func_ids` | Active problem IDs |
| `selected_alg_ids` | Active algorithm IDs |
| `cfg` | Full configuration (colors, budget, flags, plot settings) |

**Useful helpers already available:**

| Function | Purpose |
|----------|---------|
| `get_algorithm_color(name, cfg$my_colors)` | Look up algorithm color |
| `with_linetype_warning_suppressed(expr)` | Suppress plotly linetype warnings |
| `plot_violin_distribution(ds, title, file, cfg, ...)` | Reusable violin plot builder |
| `apply_uniform_axes_to_subplots(p, x_cfg, y_cfg)` | Uniform axes for subplot grids |
| `build_subplot_annotations(p, labels, font)` | Subplot title annotations |
| `tune_ecdf_xaxis(p, x_range)` | ECDF-specific x-axis tuning |

## Step 4: Wire into `run_all_benchmarks()` (`benchmark_plots.R`)

Add the call at the appropriate position:

```r
run_all_benchmarks <- function(dsl_plot_base, analysis_data, plot_data,
                               selected_func_ids, selected_alg_ids, cfg) {
  cat("\n========== BENCHMARK-PLOTS ==========\n")

  benchmark_1a_convergence_multi_func(dsl_plot_base, cfg)
  # ... existing calls ...
  benchmark_4b_quality_bars(dsl_plot_base, selected_func_ids, selected_alg_ids, cfg)

  benchmark_X_my_new_plot(plot_data, cfg)   # ← new

  cat("========== ALLE BENCHMARKS ABGESCHLOSSEN ==========\n\n")
}
```
